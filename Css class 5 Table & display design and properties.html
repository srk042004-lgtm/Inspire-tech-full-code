üé• Class 5: Table & Display Property & Design Properties

This class focuses on two critical areas: creating clean, professional **Table designs** and mastering the fundamental **`display` property**, which controls how every HTML element behaves in the layout flow. We also cover useful general **design properties** for visual flair.

üñ•Ô∏è The CSS `display` Property (Layout Control)

The **`display`** property is the most fundamental concept in CSS layout. It determines whether an element is treated as a block-level, inline-level, or a combination of both. Misunderstanding this property leads to common layout issues.

Understanding Block vs. Inline:
Value Key Characteristics Usage Context
**`block`** * **Takes up 100% width** of its parent. * Always starts on a **new line**. * **Respects** `width`, `height`, and all four `margin`/`padding` sides. Major structural elements like headers, paragraphs, lists, and containers (`div`).
**`inline`** * Only takes the width needed for its **content**. * Flows **side-by-side** with other inline content. * **Ignores** explicit `width` and `height`; vertical margins are ignored. Small pieces of content like links (`a`), emphasis (`strong`), and spans (`span`).
**`inline-block`** * Flows **side-by-side** like `inline`. * **Respects** `width`, `height`, and all `margin`/`padding` sides like `block`. Perfect for horizontal navigation menus or image cards where elements need to sit next to each other but require dimensions.
**`none`** The element is completely removed from the document layout. It takes up no space and is not visible. Used extensively for toggling hidden menus, modals, or conditionally showing content via JavaScript. 

/* Example: Making inline links stack vertically like a menu */
.nav-link {
  display: block; 
  margin-bottom: 5px;
}

/* Example: Making block elements sit horizontally */
.icon-container {
  display: inline-block;
  width: 100px; /* Now respects width */
  height: 100px;
}

üìä Comprehensive Table Styling and Design

Tables are used to display tabular data, not for general page layout (that's what Flexbox and Grid are for). CSS is vital for improving table readability and aesthetics.

Key Table Styling Techniques:
  • **`border-collapse`**: The default is `separate`, which leaves space between cells. Setting it to **`collapse`** merges borders, which is the standard modern look.
  • **Cell Padding**: Apply `padding` to `<th>` and `<td>` elements to create internal whitespace, drastically improving readability.
  • **Zebra Striping**: This technique involves using the **`:nth-child(even)`** or **`:nth-child(odd)`** pseudo-selector on `<tr>` (table row) to apply alternating row colors.
  • **Hover Effects**: Use the **`tr:hover`** pseudo-class to highlight the entire row when the mouse moves over it, helping users track data.
Example of a Styled Table:
Name Department Status
Ali Khan Marketing Active
Sana Malik Development Active
Fahad Qazi HR On Leave 

table {
  width: 100%;
  border-collapse: collapse; /* Essential for clean borders */
}

th, td {
  border: 1px solid #ccc;
  padding: 10px; /* Internal spacing */
}

tr:nth-child(even) {
  background-color: #f2f2f2; /* Zebra striping */
}

tr:hover {
  background-color: #e9e9e9; /* Row highlighting */
}

üé® General Design and Visibility Properties

These properties are powerful tools for managing the visual state and user interaction of any element.

Opacity and Visibility:
Property Purpose & Key Difference
**`opacity`** Sets the transparency level (0.0 to 1.0). An element with `opacity: 0;` is invisible but **still occupies space** and can be interacted with (e.g., clickable).
**`visibility`** `hidden` or `visible`. An element with `visibility: hidden;` is invisible, but **still occupies space** (just like `opacity: 0;`). Use `display: none;` if you want the element to occupy no space.
Aesthetic Properties:
  • **`box-shadow`**: Adds depth to elements. Syntax is typically `horizontal-offset vertical-offset blur-radius spread-radius color`.
  • **`cursor`**: Changes the mouse pointer to guide the user. Common values include `pointer` (for clickable items), `text` (for text fields), and `not-allowed`. 

.card-effect {
  /* Shadow: 0px X, 4px Y, 8px Blur, -4px Spread, Black with 25% opacity */
  box-shadow: 0 4px 8px -4px rgba(0, 0, 0, 0.25);
  cursor: pointer;
}

.warning-overlay {
  visibility: hidden; /* Takes up space, but hidden */
  opacity: 0.5;
}
