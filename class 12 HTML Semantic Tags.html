Class 14: HTML Semantic Tags

Is class me hum seekhenge ke **Semantic Tags** kya hote hain aur unka istemaal karna kyun zaroori hai. Yeh tags sirf styling ke liye nahi, balki code ko **meaningful** aur **accessible** banane ke liye hain.

Video Lesson: Semantic HTML Explained

1. Semantic Tags Ka Matlab Kya Hai?

**Non-Semantic Tags** (jaise `<div>` aur `<span>`) sirf containers hote hain. Woh bataate nahi ki unke andar ka content kya hai. Lekin **Semantic Tags** (jaise `<header>` aur `<footer>`) browser aur developer ko batate hain ke us element ka **purpose** kya hai.

Inke do bade fayde hain: **Accessibility** (screen readers ko help milti hai) aur **SEO** (search engines ko page ka structure samajhne me aasaani hoti hai).

2. Important Semantic Tags

A. Layout Tags

Tag Purpose (Kaam)
<header> Website ka introductory content. Logo, Title, aur Navigation isme aate hain.
<nav> Main navigation links ka set. Hamesha <header> me use hota hai.
<main> Webpage ka woh hissa jo **unique** aur **primary** content rakhta hai. Page me sirf ek hi <main> hona chahiye.
<footer> Copyright, Contact information, ya secondary links.

B. Content Grouping Tags

Tag Purpose (Kaam)
<section> A generic block or grouping of content, often with a heading (like a chapter).
<article> Ek independent, self-contained piece of content (e.g., a blog post, news story, comment).
<aside> Aise content ke liye jo aas-paas ke content se related ho, lekin uski main story ka part na ho (e.g., sidebars ya advertisements).
<hgroup> Headings aur sub-headings ko group karne ke liye.

C. Special Semantic Tags

Tag Purpose (Kaam)
<figure> Images, illustrations, code snippets jaise self-contained content ko wrap karta hai.
<figcaption> <figure> ke andar ka caption define karta hai.
<time> Date aur Time ko machine-readable format me show karta hai.

3. Full Semantic Page Structure

Dekhiye, ek modern webpage ka structure semantic tags ke saath kaisa lagta hai: 


<!DOCTYPE html>
<html lang="en">
<head>...</head>
<body>

    <header>
        <h1>My Awesome Website</h1>
        <nav>
            <ul><li>Links</li></ul>
        </nav>
    </header>

    <main>
        <section>
            <h2>Introduction</h2>
            <p>This is the main content area.</p>
        </section>

        <article>
            <h2>A Blog Post</h2>
            <p>Content of the post...</p>
            <aside>
                <p>Related links or author bio.</p>
            </aside>
        </article>
        
    </main>

    <footer>
        <p>&copy; 2025 All Rights Reserved</p>
    </footer>

</body>
</html>
