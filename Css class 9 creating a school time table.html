    üé® Step 2: Applying CSS for Readability

    CSS converts the raw HTML structure into a presentable timetable. We focus on four main areas to improve data comprehension:

    Styling Techniques Employed:
    • **Borders**: We use **`border-collapse: collapse;`** on the `` for crisp, single borders, and apply a subtle border to all `
      ` and `` cells.
    • **Spacing**: Generous **`padding`** on cells makes the text breathe and is crucial for legibility.
    • **Zebra Striping**: Using **`tr:nth-child(even)`** for alternating row backgrounds helps users track data across long rows.
    • **Semantic Coloring**: Assigning distinct background colors (using specific CSS classes like `.break` or `.subject`) to different types of cells (e.g., breaks, labs, lectures).
      • 
.timetable {
  border-collapse: collapse;
  width: 100%;
}
.timetable td {
  padding: 15px; /* Increase cell size for data visibility */
  text-align: center;
}
.timetable .break {
  background-color: #f7a300; 
  color: white;
  font-weight: bold;
}
.timetable .subject {
  background-color: #e6f7ff;
}

      üöÄ Step 3: Advanced Features and Accessibility

      To make the timetable modern and interactive, we add features that improve user interaction:

      • **Hover Effects**: Use **`tr:hover`** to slightly change the background color of the row currently under the mouse. This helps the user track their subject/time across the row.
      • **Cell Alignment**: Use **`vertical-align: middle;`** on `
      		` to ensure content remains centered vertically, especially when using `rowspan` or when cell heights vary.
    • **Responsiveness (Introduction)**: For a very large table, consider making the table scrollable on small screens by wrapping it in a container with **`overflow-x: auto;`**. This is a basic form of responsiveness.
