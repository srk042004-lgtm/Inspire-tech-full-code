‚ú® Class 7: Pseudo-classes, Tooltips & Hover Effects

In this crucial class, we move beyond static styling and introduce **dynamic CSS**. We will learn how to apply styles based on an element's **state** (Pseudo-classes) or **position** (Pseudo-elements), and use these concepts to build interactive features like **tooltips** and **advanced hover effects**.

üñ±Ô∏è Pseudo-classes (`:`) - Styling Element States

A **Pseudo-class** is used to define a special state of an element. It starts with a single colon (`:`). They are essential for user interaction and dynamic lists.

Common Interactive Pseudo-classes:
  • **`:hover`**: Applies styles when the user's mouse pointer is over the element (used for buttons, links, etc.).
  • **`:active`**: Applies styles when the element is being clicked.
  • **`:focus`**: Applies styles when an element (like an input field or button) has received focus, typically via clicking or tabbing.
  • **Link Pseudo-classes (L-V-H-A)**:
    • **`:link`**: Unvisited link.
    • **`:visited`**: Visited link.
Structural Pseudo-classes (Targeting Position):
  • **`:first-child`**: Targets the element if it is the first child of its parent.
  • **`:last-child`**: Targets the element if it is the last child of its parent.
  • **`:nth-child(n)`**: Targets children based on a formula (e.g., `(even)` or `(2n+1)` for odd, often used for zebra-striping). 

/* Changes button color when mouse hovers over it */
button:hover {
  background-color: #0056b3;
}

/* Styles every third list item */
li:nth-child(3n) {
  border-bottom: 2px dashed #ccc;
}

‚úçÔ∏è Pseudo-elements (`::`) - Adding Content

A **Pseudo-element** is used to style a specific part of an element. It starts with a double colon (`::`). The two most powerful are `::before` and `::after`, which allow you to insert entirely new content *without* adding extra HTML.

Key Pseudo-elements:
  • **`::before`**: Inserts content before the content of the selected element.
  • **`::after`**: Inserts content after the content of the selected element.
  • **`::first-line`**: Styles the first line of an element's text.
  • **`::selection`**: Styles the portion of an element that is selected by the user (highlighted).
**Note on `::before` and `::after`:** To make these visible, you **must** include the `content` property, even if it's set to an empty string (`content: "";`). They are typically used for decorative elements, icons, or clear fixes. 

/* Example: Adding a decorative dash before a heading */
h4::before {
  content: "‚Äî ";
  color: #28a745;
  font-weight: bold;
}

/* Example: Creating a styled, invisible overlay after an image */
.card-image::after {
  content: "";
  display: block;
  position: absolute;
  background: rgba(0, 0, 0, 0.2);
  width: 100%;
  height: 100%;
}

üí° Creating Tooltips and Custom Hover Effects

Combining **`:hover`** (a pseudo-class) and **`::after`** (a pseudo-element) is the classic pure-CSS method for creating stylish tooltips.

Tooltip Implementation Steps:
  1. Set the main container (e.g., a `span` or `div`) to **`position: relative;`**.
  2. Use the **`::after`** pseudo-element to hold the tooltip text/box.
  3. Set the `::after` element to **`position: absolute;`** to take it out of the flow.
  4. By default, set the `::after` element to `visibility: hidden;` and `opacity: 0;`.
  5. Use the **`:hover`** pseudo-class on the container to change the `visibility` and `opacity` to show the tooltip.

Creating Smooth Hover Effects:

To make transitions between states look professional (e.g., button color changing smoothly), always use the **`transition`** property. 


.button-effect {
  background-color: #0d6efd;
  transition: background-color 0.3s ease, transform 0.2s ease;
}

.button-effect:hover {
  background-color: #0a58ca; /* Darker on hover */
  transform: translateY(-2px); /* Slight lift effect */
}
