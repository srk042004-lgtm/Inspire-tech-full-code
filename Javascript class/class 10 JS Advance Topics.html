PART 1: Asynchronous JavaScript (Promises & Fetch)

JavaScript is fundamentally **single-threaded** (it executes one operation at a time). However, tasks like fetching data from a server or setting a timer can take time. If JavaScript waited for these tasks, the entire web page would freeze.

**Asynchronous** code solves this by allowing these slow tasks to run in the background while the rest of the code continues executing.

A. Promises (The Foundation)

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation and its resulting value. A Promise can be in one of three states:

  • **Pending**: Initial state, neither fulfilled nor rejected.
  • **Fulfilled**: The operation completed successfully (use `.then()`).
  • **Rejected**: The operation failed (use `.catch()`).
B. Making Network Requests with `fetch()`

The `fetch()` API is the modern way to retrieve resources (like data from an API) asynchronously. It returns a Promise.

Example: Using `fetch()` and `.then()`
// Fetch returns a response object (Promise 1)
fetch('https://api.example.com/data')
    .then(response => {
        // We must call .json() to parse the body (Promise 2)
        return response.json(); 
    })
    .then(data => {
        // Work with the actual data
        console.log("Data received:", data); 
    })
    .catch(error => {
        // Handle any error during fetch or parsing
        console.error("Fetch error:", error);
    });

PART 2: Error Handling with `try...catch`

Bugs and unexpected issues (like network errors) are inevitable. Proper error handling ensures your application doesn't crash completely when something goes wrong.

The `try...catch` Statement
  • **`try`**: Defines the block of code to be tested for errors.
  • **`catch`**: Defines the block of code to be executed if an error occurs in the `try` block.
  • **`finally`**: Defines code that is executed regardless of the result (e.g., closing a connection).
try {
    // Attempt to run code that might fail
    let result = complexFunction(); 
    console.log("Success!");
} catch (error) {
    // If complexFunction fails, the error object is caught here
    console.error("An error occurred:", error.message);
    // You could display an error message to the user here
} finally {
    console.log("This always runs.");
}

PART 3: Modern ES6+ Features

These are powerful, concise syntax features that significantly improve code readability.

A. Object and Array Destructuring

Destructuring assignment syntax is a JavaScript expression that makes it possible to unpack values from arrays or properties from objects into distinct variables.

Object Destructuring
const user = { id: 101, name: "Ali", email: "a@mail.com" };

// Traditional way
// const name = user.name;
// const id = user.id;

// Destructuring (Pulls properties out by name)
const { name, id } = user;
console.log(name); // Output: Ali
Array Destructuring
const colors = ["Red", "Green", "Blue"];

// Destructuring (Pulls values out by position)
const [c1, c2] = colors;
console.log(c1); // Output: Red
B. Spread (`...`) and Rest Operators (`...`)

These use the same syntax (`...`) but serve different purposes based on where they are used.

  • **Spread Operator**: Expands an iterable (like an array) into its individual elements. **Use case: Copying arrays, combining arrays/objects.**
  • **Rest Operator**: Collects multiple elements and "packs" them into a single array. **Use case: Function parameters.**
// 1. Spread (Used in array/object literal)
const arr1 = [1, 2];
const arr2 = [3, 4];
const combined = [...arr1, ...arr2]; // combined is [1, 2, 3, 4]

// 2. Rest (Used in function parameter)
function logArguments(first, ...restOfArgs) {
    console.log(restOfArgs); // restOfArgs is an array of remaining arguments
}
logArguments(10, 20, 30, 40); // Output: [20, 30, 40]
