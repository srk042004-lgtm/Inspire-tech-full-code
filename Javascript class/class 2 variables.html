This class covers the two pillars of foundational programming: **Variables** (the containers we use to store information) and **Data Types** (the different kinds of information those containers can hold).

PART 1: VARIABLES (`var`, `let`, `const`)

Variables are named storage locations. The keyword you use dictates the variable's **scope** (where it's accessible) and whether it can be **reassigned** (changed) or **redeclared** (defined again).

Variable Comparison:
Feature `var` (Old, Pre-ES6) `let` (Modern) `const` (Modern)
**Scope** **Function Scope** **Block Scope** **Block Scope**
**Reassignment?** Yes Yes No (Constant)
**Redeclaration?** Yes (Avoid!) No (Error) No (Error)
Understanding Scope

**Block Scope** (used by `let` and `const`) means the variable is only visible within the nearest curly braces `{}`. This prevents accidental changes and is the safer standard for modern code.

**Function Scope** (used by `var`) means the variable is visible throughout the entire function, even if declared inside a sub-block like an `if` statement or `for` loop. This leads to bugs. 

// Example of Block Scope (let/const) vs. Function Scope (var)
var globalCount = 0;

if (true) {
    let blockID = 100;
    var functionMessage = "Hello from inside block";
}

// console.log(blockID); // ERROR: Not defined (Block Scope works)
console.log(functionMessage); // Output: "Hello from inside block" (var escapes block)
**Modern Best Practice:** Use **`const`** by default. Only switch to **`let`** if you need to reassign the variable's value later. **Avoid `var`**.

PART 2: DATA TYPES

Data types tell the computer what kind of data is stored in a variable, which dictates what operations can be performed on it. JavaScript is a **dynamically typed** language, meaning you don't declare the type; the type is determined automatically when the code is executed.

I. Primitive Data Types (The Simple Types)

Primitives are simple data types that hold only a single value. There are seven primitive types:

  1. **String** (`"Hello World"`, `'Single Quote'`): Used for text. Strings must be enclosed in quotes.
    `const greeting = "Welcome";`
  2. **Number** (`10`, `3.14`): Used for both integers (whole numbers) and floating-point numbers (decimals).
    `let age = 30;`
    `let price = 19.99;`
  3. **Boolean** (`true`, `false`): Used for logical decisions. A variable can only hold one of these two values.
    `const isStudent = true;`
  4. **Undefined**: A variable that has been declared but has **not been assigned a value**.
    `let studentName; // Value is undefined`
  5. **Null**: Represents the intentional absence of any object value. It means "nothing" or "empty."
    `let car = null; // We explicitly set the value to be empty`
  6. **Symbol** (ES6+): Used to create unique identifiers (rarely used by beginners).
  7. **BigInt** (ES2020): Used for numbers larger than the standard `Number` type can safely represent.
II. Non-Primitive (Reference) Data Types (The Complex Types)

Non-primitive types are complex and can store collections of data. They are stored by **reference**, meaning the variable holds a pointer to the location in memory where the data lives, not the data itself.

  1. **Object**: The most important non-primitive type. Used to store collections of key-value pairs (properties and methods). 
    const person = {
  firstName: "Ali",
  lastName: "Khan",
  age: 25
};
  2. **Array**: A special type of object used to store ordered lists of values (which can be of any data type). 
    const colors = ["red", "green", "blue"];
  3. **Function**: Also considered a non-primitive type in JavaScript, used to execute blocks of code.
Checking Data Types

You can check the type of any variable using the built-in **`typeof`** operator. 

console.log(typeof "Hello");  // Output: "string"
console.log(typeof 123);    // Output: "number"
console.log(typeof true);   // Output: "boolean"
console.log(typeof {});     // Output: "object"
console.log(typeof []);     // Output: "object" (Note: Arrays are a type of object in JS)
console.log(typeof null);   // Output: "object" (A historical bug in JavaScript!)
**IMPORTANT NOTE:** The `typeof null` returning `"object"` is a famous historical bug in JavaScript that was never fixed to avoid breaking old websites. You must remember this exception.
