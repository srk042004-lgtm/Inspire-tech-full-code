This class introduces two core concepts: **Loops**, which are used to automate repetitive tasks, and **Strings**, the essential data type for handling text, which often requires repetitive processing.

PART 1: Loops (Automating Repetitive Tasks)

Loops execute a block of code multiple times, as long as a specified condition is true. They save countless lines of redundant code.

A. The `for` Loop (The General Purpose Loop)

The `for` loop is the most commonly used loop, especially when you know exactly how many times you want to iterate (like counting from 1 to 10, or looping through an array).

A `for` loop requires three parts, separated by semicolons: **Initialization**, **Condition**, and **Increment/Decrement**.

Syntax: `for (initialization; condition; increment)` 
// Print numbers from 1 to 5
for (let i = 1; i <= 5; i++) {
    console.log("Count: " + i);
}

// Output:
// Count: 1
// ...
// Count: 5
B. The `while` Loop (Condition-Based Loop)

The `while` loop runs only as long as its specified condition remains `true`. It is best used when you don't know the exact number of iterations beforehand (e.g., waiting for user input).

Syntax: `while (condition)` 
let fuel = 10;
while (fuel > 0) {
    console.log("Car running, fuel remaining: " + fuel);
    fuel--; // MUST include an increment/decrement to avoid infinite loop!
}
C. The `do...while` Loop

The `do...while` loop is similar to `while`, but it guarantees that the code block **executes at least once**, because the condition is checked *after* the first run.

Syntax: `do { code } while (condition)` 
let attempt = 0;
do {
    console.log("Attempting at least once.");
    attempt++;
} while (attempt < 0); 
// Loop runs once, then checks condition (0 < 0 is false), and stops.
D. Loop Control: `break` and `continue`
  • **`break`**: Immediately exits the loop entirely.
  • **`continue`**: Skips the current iteration and jumps to the next one. 
for (let i = 1; i <= 10; i++) {
    if (i === 5) {
        continue; // Skip 5, jumps to i=6
    }
    if (i === 8) {
        break;    // Stop looping when i is 8
    }
    console.log(i); // Prints 1, 2, 3, 4, 6, 7
}

PART 2: String Manipulation

A String is a sequence of characters, and it is a primitive data type. In JavaScript, strings are **immutable**, meaning once a string is created, you cannot change it; any method that appears to "change" a string actually creates and returns a **new** string.

A. String Properties and Access
  • **`.length`**: A property that returns the number of characters in the string.
  • **Indexing**: Characters are accessed using square brackets `[]`, starting from index **0**. 
let word = "Python";
console.log(word.length); // Output: 6
console.log(word[0]);     // Output: 'P' (First character)
console.log(word[5]);     // Output: 'n' (Last character)
B. Essential String Methods

String methods are functions built into the String object used to transform or analyze the text.

Method Description Example
`.toLowerCase()`
`.toUpperCase()`		 Converts the string to all lowercase or all uppercase. `"Text".toLowerCase()` returns `"text"`
`.trim()` Removes whitespace from both ends of a string. `" hi ".trim()` returns `"hi"`
`.slice(start, end)` Extracts a section of a string and returns it as a new string. (End index is non-inclusive) `"apple".slice(1, 4)` returns `"ppl"`
`.replace(old, new)` Replaces the first occurrence of a specified value with another value. `"dog".replace('d', 'f')` returns `"fog"`
`.includes(text)` Checks if a string contains another string, returning a boolean (`true`/`false`). `"Hello".includes('ell')` returns `true`
C. Template Literals (ES6)

Template literals (backticks `` ` ``) are the modern way to define strings. They allow for **string interpolation** (inserting variables directly into the string) and multi-line strings easily. 

let name = "Imran";
let subject = "JavaScript";

// Old way (Concatenation)
// let message = "Hi, " + name + ". You are learning " + subject + ".";

// New way (Template Literal)
let message = `Hi, ${name}. You are learning ${subject}.`;
console.log(message);
