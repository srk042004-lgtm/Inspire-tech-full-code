Before we interact with the web page, we must deeply understand the **Object** structure, as everything in the browser (the page, the elements, the history) is represented as an object. This leads directly into the **DOM**.

PART 1: JavaScript Objects (Key-Value Pairs)

Objects are non-primitive data types used to store collections of data and more complex entities. They store information in **key: value** pairs.

A. Object Creation and Structure

Objects are defined using curly braces `{}`. The keys are strings (but usually unquoted) and the values can be any data type, including other objects or functions (methods). 

const student = {
    // Properties (Key-Value Pairs)
    name: "Hassan",
    rollNumber: 42,
    isEnrolled: true,
    courses: ["JS", "HTML", "CSS"], // Value is an Array
    
    // Method (Key-Value Pair where the value is a function)
    greet: function() {
        return "Hello, my name is " + this.name;
    }
};
B. Accessing and Modifying Properties

There are two primary ways to access object properties:

  • **Dot Notation (`.`)**: Used when you know the property name (most common).
  • **Bracket Notation (`[]`)**: Used when the property name contains spaces or when the property name is stored in a variable.
// Access
console.log(student.name);      // Output: Hassan
console.log(student["rollNumber"]); // Output: 42

// Execute Method
console.log(student.greet());   // Output: Hello, my name is Hassan

// Modify
student.rollNumber = 43; 

// Add New Property
student.city = "Lahore";

PART 2: Introduction to the DOM

The **Document Object Model (DOM)** is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content.

When a web page is loaded, the browser creates a DOM of the page. This model is structured like a **tree**, where every HTML tag is an **Object** (called a **Node**).

  • **Document**: The entry point to the DOM tree (the `document` object).
  • **Element**: Represents HTML tags (e.g., `<div>`, `<p>`, `<h1>`).
  • **Text Node**: The actual text content inside elements.
**The Bridge:** JavaScript uses the global **`document`** object to access and modify this tree structure.

PART 3: DOM Selectors (Finding Elements)

The first step in DOM manipulation is always **selecting** (finding) the HTML element you want to work with.

A. Selecting a Single Element

These methods return the **first** matching element found in the document, or `null` if nothing is found.

1. `getElementById()` (By ID)

Used to find an element by its unique `id` attribute. This is the fastest selector.

// HTML: <h1 id="main-title">...</h1>
const title = document.getElementById("main-title");
2. `querySelector()` (By CSS Selector)

The versatile modern method. Finds the first element that matches the specified CSS selector (e.g., tag name, class, ID).

const firstParagraph = document.querySelector("p"); // Selects the first <p> tag
const myDiv = document.querySelector(".container"); // Selects the first element with class 'container'
B. Selecting Multiple Elements

These methods return a collection of all matching elements, which acts like an array and can be looped through.

1. `getElementsByClassName()` (By Class)

Returns a live HTMLCollection of all elements with a given class name.

// HTML: <li class="list-item">...</li>
const allListItems = document.getElementsByClassName("list-item");
console.log(allListItems.length); // Get the count of elements
2. `querySelectorAll()` (By CSS Selector)

Returns a static NodeList of all elements matching the CSS selector. This is the most popular modern method for multiple selection.

// Select all elements that are <li> inside a <ul>
const allNavLinks = document.querySelectorAll("ul li a");
