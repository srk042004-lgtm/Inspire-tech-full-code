The web is reactive. **Event Handling** is the mechanism that allows JavaScript to detect and respond to actions that occur on the web page, making your applications truly interactive.

PART 1: The `addEventListener()` Method

The modern, preferred way to handle events is using the `addEventListener()` method. It allows you to attach multiple handlers to a single element without overwriting previous ones.

Syntax: `element.addEventListener(event, function, useCapture)`
  • **`event`**: A string specifying the event type (e.g., `'click'`, `'mouseover'`).
  • **`function`**: The function to run when the event occurs (the **Event Handler**).
  • **`useCapture`**: An optional boolean (usually `false`). 
// Assume HTML: <button id="myBtn">Click Me</button>
const button = document.getElementById("myBtn");

// Define the function to execute
function handleClick() {
    alert("Button was clicked!");
    button.textContent = "Clicked!";
}

// Attach the listener
button.addEventListener('click', handleClick);

// You can attach another function to the same event!
button.addEventListener('click', () => {
    console.log("Second listener fired.");
});

PART 2: Common Event Types

Events fall into various categories, based on the type of user interaction.

Category Event Type (String) Description
**Mouse Events** `'click'`, `'mouseover'`, `'mouseout'` A mouse button was clicked, cursor moved over an element, cursor moved away.
**Keyboard Events** `'keydown'`, `'keyup'` A key is pressed down, a key is released.
**Form Events** `'submit'`, `'input'`, `'change'` A form is submitted, value of an input changes, element's value is committed.
**Document/Window Events** `'load'`, `'resize'`, `'scroll'` Page finishes loading, browser window size changes, user scrolls.

PART 3: The Event Object

When an event fires, the browser automatically creates an **Event Object** and passes it as the **first argument** to your handler function. This object contains vital information about the event itself.

Example: Using the Event Object (`e` or `event`) 
document.addEventListener('keydown', (e) => {
    // e.key: The actual key pressed (e.g., 'A', 'Enter', 'Shift')
    console.log("You pressed the key: " + e.key);
    
    // e.target: The element that triggered the event (the document in this case)
    console.log("Event target tag: " + e.target.tagName); 
});
A. Key Properties in the Event Object
  • **`e.target`**: The DOM element that triggered the event (e.g., the button that was clicked).
  • **`e.preventDefault()`**: A method used to stop the browser's default action for certain events (critical for form submissions and link clicks).
  • **`e.stopPropagation()`**: A method used to stop the event from bubbling up the DOM tree (see next section).
Example: Preventing Default Action 
// HTML: <a href="https://google.com" id="link">Go to Google</a>
const link = document.getElementById("link");

link.addEventListener('click', (e) => {
    e.preventDefault(); // Stops the browser from navigating to google.com
    alert("Navigation prevented!");
});

PART 4: Event Bubbling and Capturing

When an event occurs on an element, it doesn't just happen there; it propagates (travels) up and down the DOM tree.

  • **Capturing Phase**: The event travels *down* from the `window` to the target element.
  • **Target Phase**: The event is handled at the target element.
  • **Bubbling Phase (Default)**: The event travels *up* from the target element back to the `window`.
**Event Delegation:** This is a powerful technique where you attach a single event listener to a parent element instead of attaching listeners to every single child element. The event object's `e.target` property is then used to determine which child actually triggered the event.
