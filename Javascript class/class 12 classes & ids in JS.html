üí° DOM Reference: Managing Classes and IDs

In professional web development, the best way to change the appearance of an element is usually **not** by setting individual styles (`.style.color`), but by toggling CSS **Classes**. JavaScript also uses **IDs** heavily for unique selection.

PART 1: Managing IDs in JavaScript

In HTML, an `id` must be unique per page. In JavaScript, the `id` property on an element object is used for both reading and writing the element's ID.

A. Selecting by ID
// HTML: <button id="submitBtn">...</button>
// The fastest way to select a single, unique element:
const button = document.getElementById("submitBtn");
B. Reading and Changing ID

You can easily check the current ID or assign a new one, though changing an ID is rare in practice. 

// Check the current ID
console.log(button.id); // Output: "submitBtn"

// Change the ID (use with caution!)
button.id = "newSubmitId";

PART 2: Controlling CSS Classes with `classList`

The `classList` property of a DOM element is the key to dynamic styling. It returns a collection of the element's class attributes and provides methods to easily modify them.

A. Adding, Removing, and Toggling Classes

These are the three most frequently used methods for controlling dynamic appearance.

Example Setup (HTML & CSS)
<!-- HTML -->
<div id="message" class="base-style">Status Update</div>

/* CSS */
.base-style { border: 2px solid grey; }
.highlight { background-color: yellow; }
.error { color: red; font-weight: bold; }
JavaScript ClassList Methods
const message = document.getElementById("message");

// 1. ADD: Add a class
message.classList.add("highlight"); 
// Result: class="base-style highlight"

// 2. REMOVE: Remove a class
message.classList.remove("base-style");
// Result: class="highlight"

// 3. TOGGLE: Add the class if it's not present, remove if it is (great for buttons)
message.classList.toggle("error"); // Adds 'error'
message.classList.toggle("error"); // Removes 'error'
B. Checking and Replacing Classes

Two less frequent but useful methods for more complex state management. 

// 4. CONTAINS: Check if a class exists (returns boolean: true/false)
if (message.classList.contains("highlight")) {
    console.log("Element is highlighted."); 
}

// 5. REPLACE: Replace an existing class with a new one
// message.classList.replace("oldClass", "newClass");
message.classList.replace("highlight", "warning");
