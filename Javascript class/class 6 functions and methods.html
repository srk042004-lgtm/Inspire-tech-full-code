üì¶ Class 6: Functions, Methods & Scope

**Functions** are the core of all organized JavaScript programming. They allow you to bundle a block of code, give it a name, and reuse it throughout your application. They are essential for writing clean, efficient, and readable code.

PART 1: Function Fundamentals

A. Function Declaration and Calling

A function is defined using the `function` keyword, followed by the function name, a list of parameters (if any), and the function body enclosed in curly braces.

Syntax: Function Declaration
// 1. Function Declaration (Definition)
function greetUser() {
    console.log("Welcome to Inspire Tech Academy!");
}

// 2. Function Call (Execution)
greetUser(); // Output: Welcome to Inspire Tech Academy!
B. Parameters and Arguments
  • **Parameters**: The names listed in the function definition (like local variables waiting for input).
  • **Arguments**: The actual values passed to the function when it is called.
Example: Parameters and Arguments
// 'num1' and 'num2' are parameters
function addNumbers(num1, num2) {
    let sum = num1 + num2;
    console.log(sum);
}

// 5 and 3 are arguments
addNumbers(5, 3); // Output: 8
C. The `return` Keyword

The `return` statement immediately stops function execution and returns a value back to the place where the function was called. If you don't use `return`, the function returns `undefined` by default.

Example: Returning a Value
function calculateArea(length, width) {
    let area = length * width;
    return area; // Sends the 'area' value back
}

// The returned value (20) is stored in the variable 'roomArea'
let roomArea = calculateArea(4, 5);
console.log(roomArea); // Output: 20

PART 2: Arrow Functions (Modern Syntax)

Introduced in ES6, **Arrow Functions (`=>`)** offer a concise, cleaner syntax for writing function expressions. They are now the standard way to write functions in modern JavaScript, especially for shorter operations.

Comparison: Traditional vs. Arrow
TypeSyntax
**Traditional** `function isEven(num) { return num % 2 === 0; }`
**Arrow (Full)** `const isEven = (num) => { return num % 2 === 0; };`
**Arrow (Concise/Implicit Return)** `const isEven = num => num % 2 === 0;` Best for simple code
Concise Arrow Function Rules:
  • If there is only **one parameter**, you can omit the parentheses around it (`num => ...`).
  • If the function body is a **single expression**, you can omit the curly braces `{}` and the `return` keyword (Implicit Return).
// Zero parameters require parentheses
const sayHi = () => console.log("Hi");

// One parameter, no parentheses needed
const double = x => x * 2;
console.log(double(10)); // Output: 20

PART 3: Functions, Methods & Scope

A. The Difference: Function vs. Method

Technically, a **Method** is just a function that is stored as a property of an **Object**.

  • **Function**: Exists independently (e.g., `greetUser()`, `addNumbers(a, b)`).
  • **Method**: Is called on an object using dot notation (e.g., `students.push('Ali')`, `myString.toUpperCase()`).
Example: Method
// 'person' is an Object
const person = {
    firstName: "Sarah",
    // 'introduce' is a Method
    introduce: function() {
        console.log("My name is " + this.firstName);
    }
};

person.introduce(); // Output: My name is Sarah
B. Function Scope and Variable Access

Functions create their own local scope. Variables defined inside a function (including parameters) are **Local** and cannot be accessed from outside the function.

Variables defined outside any function are **Global** and can be accessed and modified by any function. 

let globalVar = 50; // Global Variable

function updateVar() {
    let localVar = 10; // Local Variable
    globalVar = globalVar + localVar;
    console.log(globalVar); // Can access both
}

updateVar(); // Output: 60 (50 + 10)

// console.log(localVar); // ERROR: localVar is not defined outside the function
**Best Practice:** Always try to use local variables within functions and pass necessary data in through parameters. Minimizing the use of global variables prevents conflicts and makes your code easier to debug.
